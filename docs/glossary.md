# Словарь Терминов MeetingAgent

Обновлено: 2026-05-07.

Этот словарь нужен для трех задач:

1. чтобы документация использовала одни и те же термины;
2. чтобы prompts и классификаторы понимали проектные сокращения;
3. чтобы live/offline-транскрибация могла получать `initial_prompt` с важными словами и меньше коверкала аббревиатуры.

## Проектные Термины

| Термин | Значение |
| --- | --- |
| АСУ | Автоматизированная система управления. В MeetingAgent это предметная область проектных документов. |
| ФТТ | Функционально-технические требования проекта АСУ. Не путать с `FTT-MA-*`, которые описывают требования к MeetingAgent. |
| `FTT-MA-*` | Пункты функционально-технических требований самого продукта MeetingAgent. |
| ПМИ | Программа и методика испытаний. Документ для проверки и приемки системы. |
| Паспорт ИС | Сдачный документ с описанием информационной системы, ее состава, назначения, архитектуры и эксплуатационных сведений. |
| ЦТА | Проектный термин из корпуса АСУ. Точное значение должно подтверждаться источниками RAG, а не догадкой. |
| Сдачная документация | Комплект документов, которые нужны для передачи или приемки результата проекта. |
| Проектное решение | Документ или фрагмент, который описывает выбранный способ реализации, архитектуру, интеграции и ограничения. |
| Архитектурная записка | Черновой или формальный документ с архитектурными решениями, компонентами и связями. |

## Термины MeetingAgent

| Термин | Значение |
| --- | --- |
| RAG | Поиск по локальным источникам с последующей генерацией ответа на основе найденных фрагментов. |
| Источник | Документ, транскрипт, протокол, memo или другой материал, из которого берется информация. |
| Chunk | Фрагмент текста, который индексируется и ищется через embeddings. |
| Embedding | Векторное представление chunk или вопроса. В текущем проекте используется `bge-m3`. |
| Embeddings cache | `data/embeddings_cache.jsonl`; самый дорогой по времени артефакт, который нельзя удалять без причины. |
| Numpy-индекс | Основной локальный поисковый индекс в `data/numpy_index`, собранный из chunks и embeddings cache. |
| Карточка встречи | Папка и `meeting.json`, которые объединяют запись, транскрипт, memo, протокол, задачи, решения, риски и классификацию. |
| Memo | Краткая суть встречи: что обсуждали, к чему пришли, что важно помнить. |
| Протокол | Более формальная структура встречи: вопросы, обсуждение, решения, поручения, открытые вопросы. |
| Трассировка источников | Связь ответа, решения или документа с исходным файлом, chunk или сегментом транскрипта. |
| Инкрементальное обновление | Обновление RAG только по новым, измененным или удаленным материалам. |

## Термины Live-Транскрибации

| Термин | Значение |
| --- | --- |
| MIC | Отдельная дорожка микрофона пользователя. |
| SYS | Отдельная дорожка системного звука через loopback. |
| MIX | Производный смешанный поток для быстрого просмотра; не должен быть единственным источником для хранения. |
| Segment | Фрагмент транскрипта с `source`, `start`, `end`, `text`. |
| VAD | Voice activity detection. Для live MVP выключен по умолчанию, чтобы таймкоды были предсказуемее. |
| Diarization | Разделение речи по спикерам. Не входит в MVP, но должна быть сохранена возможность добавить ее позже. |

## Черновик `initial_prompt` Для Whisper

Этот текст можно использовать как основу для `initial_prompt` в faster-whisper:

```text
Встреча по проекту АСУ. Возможные термины: ФТТ, ПМИ, Паспорт ИС, проектное решение, архитектура, ЦТА, интеграция, сдачная документация, протокол, требования, риски, задачи, заказчик, аналитик, архитектор, руководитель проекта.
```

Список нужно расширять после первых реальных транскриптов: добавлять названия модулей, систем, интеграций и устойчивые аббревиатуры из проекта.
