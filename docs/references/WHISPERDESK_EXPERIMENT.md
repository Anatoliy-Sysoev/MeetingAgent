# WhisperDesk Как Экспериментальный Референс

Обновлено: 2026-05-07.

Источник эксперимента: `C:\Users\Сотрудник\Desktop\AI\WhisperDesk`.

Важно: WhisperDesk не считается рабочей основой MeetingAgent. Это эксперимент, из которого можно брать проверенные идеи и параметры, но не переносить монолитную реализацию как production-код.

## Что Внутри Эксперимента

В `whisperdesk.py` реализован локальный Tkinter-инструмент для:

- live-транскрибации встречи;
- транскрибации готового аудио/видео файла;
- выбора микрофона;
- захвата системного звука через WASAPI loopback;
- сохранения transcript в TXT, JSONL и опционально SRT;
- сохранения `session.json` с параметрами сессии.

Используемые библиотеки:

- `faster_whisper`;
- `sounddevice`;
- `pyaudiowpatch`;
- `numpy`;
- `tkinter`.

## Что Берем В MeetingAgent

### 1. CPU-First Профиль Live

Для первого MVP live-транскрибации берем консервативный профиль:

- модель: `small`;
- compute: `int8`;
- язык: `ru`;
- chunk: 5 секунд;
- beam: 3;
- VAD: выключен по умолчанию для live-таймкодов.

Почему: этот профиль лучше подходит для ноутбука без локального GPU и дает предсказуемую задержку.

### 2. Два Источника Аудио

Берем идею раздельного выбора:

- микрофон;
- системный звук через loopback;
- смешанный поток, когда нужны оба источника.

Для MeetingAgent это должно стать частью `FTT-MA-10` и `FTT-MA-11`.

### 3. Сессионная Модель

Берем подход "одна встреча - одна папка":

```text
meeting_session/
  session.json
  live_transcript.txt
  live_segments.jsonl
  live_subtitles.srt
```

В MeetingAgent эта идея будет расширена до карточки встречи:

```text
meetings/
  YYYY-MM-DD/
    meeting_slug/
      source/
      transcript/
      artifacts/
      exports/
      meeting.json
```

### 4. JSONL Сегменты

Берем формат сегментов как основу:

```json
{"source":"MIX","start":0.0,"end":4.8,"text":"...","ts":"2026-05-07T10:00:00"}
```

Для MeetingAgent к нему позже добавятся:

- `speaker`;
- `confidence`;
- `language`;
- `meeting_id`;
- ссылки на связанные решения, задачи и ФТТ.

### 5. Start/Stop И Device Refresh

Берем пользовательский сценарий:

- обновить список устройств;
- выбрать микрофон и system audio;
- нажать старт;
- видеть live log;
- нажать стоп;
- открыть папку сессии.

В MeetingAgent это сначала может быть CLI/prototype, потом UI.

## Что Не Берем

- Не переносим Tkinter-монолит как основную архитектуру продукта.
- Не копируем build/dist/venv из WhisperDesk.
- Не считаем SRT обязательным форматом для MVP.
- Не делаем live-транскрибацию единственным способом обработки: offline-записи остаются основным надежным сценарием.
- Не смешиваем экспериментальную папку WhisperDesk с репозиторием MeetingAgent.

## Как Переносить Идеи В Продукт

Правильный путь:

1. Описать требования в `docs/product/PROJECT_STAGES_AND_FTT.md`.
2. Создать отдельный модуль live audio capture в MeetingAgent.
3. Создать отдельный writer для TXT/JSONL/SRT.
4. Подключить live-сессию к общей карточке встречи.
5. После остановки live-сессии запускать общий pipeline memo/protocol/classification.

## Минимальная Приемка Для Live MVP

Live MVP считается готовым, если:

- работает запись микрофона;
- работает запись system audio, если на машине доступен loopback;
- каждые несколько секунд появляются сегменты transcript;
- сегменты пишутся в `live_segments.jsonl`;
- есть читаемый `live_transcript.txt`;
- `session.json` хранит настройки;
- после остановки сессия может быть обработана как обычная встреча MeetingAgent.
