# Product Documentation — Project Knowledge Bot

Обновлено: 2026-05-16.

## Назначение

Эта папка содержит продуктовую документацию Project Knowledge Bot как самостоятельного продукта.

Документы раздела нужны для:

- подготовки подпроекта к выделению в отдельный репозиторий;
- объяснения пользовательской ценности;
- фиксации целевых пользователей и сценариев;
- связи MVP, требований, архитектуры и roadmap;
- отделения продуктового слоя от runtime-инструкций и кода.

## Краткое описание продукта

Project Knowledge Bot — локальный project-only AI-ассистент для команд, работающих с большим комплектом проектной и технической документации информационной системы.

Бот решает задачу быстрого поиска подтвержденной информации и подготовки ответов по источникам.

Ключевые свойства:

```text
локальная обработка
project-only режим
pre-retrieval guardrails
hybrid retrieval
ответы с citations
структурная валидация ответа
логирование запусков
baseline evaluation
```

## Статус продукта

```text
MVP Search API: готов
MVP Chat API: готов с ограничениями
QH-1 Observability + Eval Baseline: реализован, ожидает baseline-прогон
QH-2 Source Quality Filter: открыт
QH-3 Parent Expansion: открыт после QH-2 при необходимости
UI: не начат
Enterprise hardening: не начат
```

## Пользовательская ценность

Для системного аналитика:

- быстро найти подтверждение в документации;
- сопоставить требования, архитектуру, проектные решения и интеграции;
- подготовить черновик ответа с источниками;
- снизить риск галлюцинаций по памяти или общим знаниям LLM.

Для архитектора:

- быстро найти описание компонентов, потоков, интерфейсов и ограничений;
- проверить связность между архитектурой, интеграциями и требованиями.

Для тестировщика:

- найти требования и сценарии;
- понять ожидаемое поведение системы;
- сопоставить требования с тестовыми сценариями.

Для руководителя проекта:

- быстро получить статус по документации;
- увидеть ограничения и открытые вопросы;
- контролировать трассируемость ответа до источников.

## Продуктовые принципы

```text
source-first, not model-first
локальная обработка по умолчанию
ответ только при наличии источников
отказ лучше неподтвержденного ответа
качество измеряется baseline cases
UI не раньше стабильного quality baseline
```

## Что не входит в продуктовый MVP

```text
универсальный чат без источников
ответы по интернету
многопользовательский портал
RBAC по источникам
fine-tuning
LLM-as-judge как runtime dependency
NLI groundedness hard-fail
agentic tool-use
```

## Критерии перехода к следующему продуктовому этапу

Переход к UI или выделению в отдельный репозиторий возможен после:

```text
локального успешного QH-1 baseline-прогона
анализа failures
актуализации eval cases
решения по QH-2 source quality filter
синхронизации README/architecture/mvp/roadmap/runbook
удаления или архивирования устаревших активных документов
```
