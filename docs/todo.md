## Известные Риски

- Полная RAG-сборка долгая и зависит от стабильности Ollama.
- ChromaDB локально нестабилен на загрузке HNSW-индекса, поэтому не должен быть критическим dependency для query.
- У нескольких spreadsheet-файлов есть ошибки extraction из-за нестандартных stylesheets.
- Качество query теперь нужно проверять через отдельный evaluation-набор на numpy backend.
- Сгенерированные документы требуют строгого ревью источников.
